Unit 1 Overview

Resources: scratchpad.io

World wide web: collection of HTML documents (made up of HTML docs). Basis of all webpages. Glues everything together via hyperlinks aka links.
Invented in 1990, billions of pages
HTML is main type of content, but any filetype can exist on the web.

Major pieces

You -> computer -> browser -> Internet -> servers
HTTP simple protocol between you, network, server

HTML

Heart of the web, made up of these documents
Made up of text content
Markup (what it looks like)
References to other documents (images, videos, etc)
Links to other pages

HTML markup

Tag < > content < /> - opening tag and closing tag
Entire item is an “element”

Tags

bold: < b >contents< /b >
emphasis/italics: < em >contents< /em >

HTML Attributes

Attributes have values - href is an example
Tags can have multiple attributes
Example: anchor: < a href=”www.reddit.com” >link< /a >

Another tag

image: < img >
example: < img src=”url” alt=”text” >
alt is “required” - what will display if image can’t load
image is a “void” tag, since it has no content - no closing tag needed

Whitespace

text on multiple lines - all are on single lines
< br > tag stands for break (void tag) - display multiple lines
< p >content< /p > - render as one paragraph
why both < br > and < p >? < br > is inline - simply ends a line. < p > is a block - can have height and width (creates an invisible box)
< b >, < em >, < img > are also inline elements
inline typically doesn’t have other characteristics (height & width, etc)

Span and Div

< span > is inline - < span class=”foo” >text< /span >
< div > is block - < div class=”bar” >

Complete HTML

Doctype: HTML (much simpler since HTML 5)
< html > surrounds entire document
< head > metadata - includes javascript and css
< title > title of the page (displays in browser window)
< body > contents of the document (where we’ve been living)
